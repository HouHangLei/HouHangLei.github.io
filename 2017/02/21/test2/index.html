经过了昨天踩的大坑小坑，终于搭建好了hexo，域名用的是github的。我的博客地址。

本文主要记录配置过程中所参考的文章以及遇到的问题和最后的解决方案，方便日后查询整理。

基础准备

下载Node.js用来生成静态页面

到Node.js官网下载,有一个推荐用于大多数用户使用和一个包含最新功能的版本，我下载的是推荐用于大多数用户的v6.9.5版本。按照流程安装。

安装Git

安装Xcode就自带Git了，或者去这里下载，安装完成后，打开终端，输入git –version，能查看到版本号就说明安装成功了

创建reposity

在GitHub上创建reposity（没有Github的去注册一个）,注意，Repository name 必须是your_name.github.io,比如我的是HouHangLei.github.io

安装Hexo

上面的安装完以后，开始安装Hexo。

终端输入npm install -g hexo 执行

如果安装失败，终端重新输入sudo npm install -g hexo 执行

安装成功后，初始化Hexo

首先创建一个文件夹，如my_hexo.然后使用终端命令cd进入my_hexo文件夹

然后执行init命令初始化hexo

至此，全部安装工作已经完成！blog就是你的博客根目录，所有的操作都在里面进行。

生成静态页面

hexo generate（hexo g也可以）

本地启动

启动本地服务，进行文章预览调试，命令：

hexo server

浏览器输入http://localhost:4000，没有问题的话就可以看到效果了

常用命令

常用命令有四个，全部的指令可以去Hexo官网查看

1.generate 生成静态文件

2.server 启动服务器（成功后可以查看本地的更改后的效果http://localhost:4000/）

3.deploy 部署网站。部署网站前，需要预先生成静态文件（hexo g）

4.clean 清除缓存文件（db.josn）和已生成的静态文件（publick）。(如果部署后没有发生改变，可以下面命令清除缓存后，就可以看到效果了)

Hexo主题

使用其他Hexo主题

使用Hexo更换主题还算方便，先使用克隆命令安装好主题，然后更改博客的配置文件_config.yml里的theme：对应你的主题名就好了。主题的选择你可以到Hexo官方主题页面选择自己喜欢的主题。还有wiki里面的主题也可以。

你也可以到github下载页下载下来后，将文件放到themes文件夹内。不过记得把文件名改成你选择的主题名，因为下载下来的文件名和主题名会有不一样。

写文章

使用hexo new “文件名”指令新建一篇文章（官网写作指令链接）

执行完成后会在source -> _posts文件夹下多一个 文件名.md的文件。文本编辑器可以使用xcode或者MacDown都可以。MacDown语法说明

　　想要在文章中插入图片的话，可以按照Markdown语法来插入，格式为 ! [图片名称] (图片地址)。

图片的存放有两种方式：在本地my_hexo\source目录下新建一个存放图片的文件夹，比如images，然后把想要插入的图片放在里面，插入图片的路径；第二种方法是把图片上传到网络，然后插入图片路径。

第一种方式我的图片没有加载出来，可能是我写的路径有问题。我现在用的是第二种方式，将图片上传到七牛云上。这样也可以避免本地图片不小心删除等问题。
